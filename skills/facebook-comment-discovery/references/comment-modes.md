# Comment Modes

Use this before drafting any Facebook reply.

The goal is to avoid one generic polished reply style.

## 1. Quick validation

Use when the commenter is exposing a real pain or sharing a lived problem.

Write like this:

- acknowledge the pattern
- add one grounded observation
- keep it brief

Do not:

- start with generic praise
- turn the reply into a framework dump
- hijack the thread with a pitch

## 2. Direct answer

Use when the commenter is asking what to do, what tool to use, or how to
handle something.

Write like this:

- answer the question
- give one or two criteria or heuristics
- stay concrete

## 3. Light diagnostic

Use when the commenter is describing something broken or manual.

Write like this:

- narrow the issue
- suggest what to compare or test next
- avoid broad thought leadership

## 4. Recommendation response

Use when the commenter is explicitly asking for options.

Write like this:

- define the criteria first
- mention where an option fits and where it breaks
- stay out of pitch mode unless the user explicitly wants commercialization

## Anti-patterns

- do not reply just to be seen in a large creator thread
- do not sound like a consultant barging into the comments
- do not use em dashes, semicolons, or dense phrasing
- do not explain your drafting process
- do not force a GTM frame onto unrelated creator content
